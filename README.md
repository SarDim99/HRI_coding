# Alpha Mini Chatbot — DLD Conversation & Guessing Game

A spoken-dialogue application for the **Alpha Mini** social robot, designed to interact with a child who has **Developmental Language Disorder (DLD)**. The robot first holds a friendly *getting-to-know-you* conversation, then plays a personalised *word-guessing game* built around the child's interests. It listens via speech-to-text (STT), sends the recognized text to OpenAI's **GPT-4o-mini**, and speaks each reply back through text-to-speech (TTS), performing a fitting body movement with every turn.

---

## Features

- Verbal turn-taking dialogue between the robot and a child user.
- Cloud STT through `rie.dialogue.stt.stream` and TTS through `rie.dialogue.say`.
- LLM responses generated by **OpenAI GPT-4o-mini** through the official `openai` Python SDK.
- **Two conversation phases:**
  - **Introduction**---the robot learns the child's name, age, and at least one thing they like.
  - **Guessing game**---triggered once enough is known and the child agrees to play.
- **Persistent child profile:** name, age, and likes are saved to `patient_profile.json`. On the next run the robot recognizes a returning child and welcomes them back by name.
- **Personalized word-guessing game:**
  - The target word is themed around the child's interests (but never the exact word they named).
  - Three clues form a *cueing hierarchy*---broad semantic -> specific feature -> phonemic (first sound).
  - A separate LLM call **judges** each guess as `correct`, `incorrect`, or `stop`.
  - After each round the robot **asks the child** whether they want to try another word.
- **Animations on every turn:** the model picks a movement keyword (`wave`, `applause`, `think`, `shrug`, `bow`, `celebrate`, `yes`, `no`) which maps to a Blockly behavior or a custom head-motor nod/shake.
- A carefully designed **system prompt** that enforces:
  - Peer-role tone (friend, not teacher),
  - Short, simple sentences (≤ 12 words),
  - Recasting instead of "no/wrong",
  - Brief 1–2 sentence replies,
  - One question at a time, always ending with a question to keep the conversation going.
- **Self-hearing prevention:** the microphone is muted for the estimated duration of the robot's own speech, so it never transcribes its own voice.
- Stand-up at the start of the session and crouch at the end.
- Way to end: The user includes "Bye" in his response.

## Requirements

### Hardware
- An Alpha Mini robot, switched on and connected to the Robots-in-de-Klas platform.
- Pause the default agent on the robot before running the script (open the robot's menu in the portal and click *pause*. The robot will say *"agent is paused"*).

### Software
- **Python 3.10 or higher** (the script uses `list[dict]` type hints).
- An **OpenAI API key**. You can create one at <https://platform.openai.com/api-keys>.
- A **Mistral API key**. *Only if you need to test and edit changing the robot's voice*. You can create one at <https://admin.mistral.ai/organization/api-keys>.
- The realm string of your specific Alpha Mini (e.g. `rie.69f3564c26d8af1680827d60`). Found on the robot's page in the Robots-in-de-Klas portal.

### Python packages (see `requirements.txt`)
- `autobahn[twisted,serialization]`
- `pyopenssl`
- `alpha_mini_rug`
- `openai`
- `dotenv`
- `mistralai`
- `service_identity`

## Installation

### 1. Get the project

Clone the repository:

```shell
git clone https://github.com/SarDim99/HRI_coding.git
```

Or place `test.py` and `requirements.txt` together in a folder. Then open a terminal in that folder.

### 2. (Recommended) Create and activate a virtual environment

**Windows (PowerShell):**
```powershell
python -m venv venv
venv\Scripts\Activate.ps1
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install the dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## Configuration

### 1. API keys

Create a file called **`.env`** next to `test.py`:

```
OPENAI_API_KEY=sk-your_real_openai_key_here
MISTRAL_TTS_KEY=your_mistral_key_here
```

`load_dotenv()` at the top of the script picks these up. The `OPENAI_API_KEY` is required. The `MISTRAL_TTS_KEY` is only needed if you enable the experimental Mistral voice (disabled by default).

### 2. Robot realm

In `test.py`, near the bottom, replace the placeholder realm with the realm of your robot:

```python
wamp = Component(
    transports=[{
        "url": "ws://wamp.robotsindeklas.nl",
        "serializers": ["msgpack"],
        "max_retries": 0
    }],
    realm="rie.XXXXXXXXXXXXXXXXXXXXXXXX",  # <-- your robot's realm
)
```

### 3. (Optional) Change the language

The script speaks English by default. To switch to Dutch, change the language code from `"en"` to `"nl"`:

```python
yield session.call("rie.dialogue.config.language", lang="en")  # change "en" to "nl"
```

Note that the prompts are written in English, so for a Dutch session you should also translate `ROLE_PROMPT`, `INTRO_PROMPT`, and the game prompts.

### 4. (Optional) Reset the saved profile

To start fresh with a new child, delete `patient_profile.json` (or empty its contents). The robot will then run the introduction phase from scratch instead of welcoming a returning child.

## Running the demo

1. Make sure the robot is on, online (green check in the portal), and that its default agent is paused.
2. Activate your virtual environment if you created one.
3. Run the script:

```bash
python test.py
```

If everything is configured correctly, the robot will:
1. Stand up and greet the user.
2. Hold an introduction conversation, learning (or recalling) the child's name, age, and interests.
3. Suggest the guessing game; once the child agrees, give clues and judge their guesses.
4. After each word, ask whether the child wants to play another round.
5. Continue until the user says `quit` / `exit` / `:q`, or the model includes **"Bye"**, after which the robot says goodbye and crouches.

You can follow the session in the terminal, which logs the recognised speech, the chosen target word, the clue progression, and each guess verdict (lines prefixed with `[GAME]`).

## How it works

| Section | Purpose |
|---|---|
| `load_dotenv()` | Loads `OPENAI_API_KEY` (and optional `MISTRAL_TTS_KEY`) from a local `.env` file. |
| `ROLE_PROMPT` | Long-form instruction defining the robot's persona and the DLD-friendly speaking rules applied in every phase. |
| `INTRO_PROMPT` | Phase-specific prompt for the introduction; switches to a "welcome back" variant when a saved profile exists. |
| `ANIMATION_PROMPT` | Tells the model to reply with one JSON object: `{text, animation, profile}`. |
| `load_profile()` / `save_profile()` | Read and merge the child's name, age, and likes to/from `patient_profile.json`. |
| `ANIM_BEHAVIORS`, `NOD_FRAMES`, `SHAKE_FRAMES` | Map animation keywords to Blockly behaviours and custom head-motor keyframes. |
| `generate_round()` | Asks the model for a themed target word plus a 3-clue cueing hierarchy, avoiding previously used words. |
| `judge_answer()` | Classifies the child's guess as `correct`, `incorrect`, or `stop`. |
| `start_round()` / `handling_game()` | Manage game state: deliver clues one at a time, reveal the answer when clues run out, and ask to play again. |
| `ask_replay()` / `wants_replay()` | Ask the child whether they want another word and interpret their yes/no answer. |
| `handle_intro()` | Runs the introduction phase and transitions to the game on the `[GAME_START]` marker. |
| `ask_llm(user_text)` | Routes each turn to the right handler (replay / game / intro) and stores the reply in `conversation_history`. |
| `asr(frames)` | STT callback. On a `final` frame it stores the recognised text and sets `finish_dialogue`. |
| `main(session, details)` | Sets language, stands the robot, greets the user, opens the STT stream, and runs the dialogue loop. |
| `wamp = Component(...)` + `run([wamp])` | Standard Autobahn/Twisted setup that connects to the Robots-in-de-Klas WAMP server. |

## Troubleshooting

- **The robot does not say anything at startup.** Check that it is online, that the default agent is paused, and that the realm string is copied in full.
- **`openai.AuthenticationError` or `Incorrect API key provided`.** Check that you are using the correct API key in `.env`.
- **`ModuleNotFoundError: dotenv`.** Install `python-dotenv`: `pip install python-dotenv`.
- **`msgpack` error.** Run `pip install msgpack`.
- **`TLS failure: certificate verify failed`.** Run `pip install --upgrade pyopenssl`.
- **The game never starts.** The model did not emit the `[GAME_START]` marker. Make sure the child clearly agrees to play; check the terminal for the `[GAME_START] marker detected` line.
- **The same word keeps coming up.** Used words are stored in game state for the session; delete `patient_profile.json` between very short test runs if the interest list is too small to vary.
- **The robot greets a stranger by the wrong name.** A previous child's profile is still saved. Delete or clear `patient_profile.json`.
- **Conversation never ends.** The model has not produced a "Bye" and the user has not said `quit` / `exit` / `:q`.
