# Alpha Mini Chatbot — DLD Conversation & Three Games

A spoken-dialogue application for the **Alpha Mini** social robot, designed to interact with a child who has **Developmental Language Disorder (DLD)**. The robot first holds a friendly *getting-to-know-you* conversation, then plays one of **three personalised games** built around the child's interests. It listens via speech-to-text (STT), sends the recognized text to OpenAI's **GPT-4o-mini**, and speaks each reply back through text-to-speech (TTS), performing a fitting body movement with every turn.

This is the multi-game version. The entry point is **`main.py`**.

---

## The three games

| Game | Marker | Module | What the child does |
|---|---|---|---|
| **Word-guessing** | `[GUESSING_GAME_START]` | built into `main.py` | Guesses a hidden word from a 3-clue cueing hierarchy. |
| **Category naming** | `[CATEGORY_GAME_START]` | `category_game.py` | Names as many things as they can in a given category. |
| **Flash card (animals)** | `[FLASH_GAME_START]` | `flash_game.py` + `object_detect.py` | Holds up a physical animal **card** to the webcam; the robot recognizes it with a camera + YOLO. |

During the introduction the model decides which game to launch by appending the matching marker to its reply once the child agrees to play.

---

## Features

- Verbal turn-taking dialogue between the robot and a child user.
- Cloud STT through `rie.dialogue.stt.stream` and TTS through `rie.dialogue.say`.
- LLM responses generated by **OpenAI GPT-4o-mini** through the official `openai` Python SDK.
- **Conversation phases:**
  - **Introduction** — the robot learns the child's name, age, and at least one thing they like.
  - **Game** — one of the three games above, triggered once enough is known and the child agrees to play.
- **Persistent child profile:** name, age, and likes are saved to `patient_profile.json`. On the next run the robot recognizes a returning child and welcomes them back by name.
- **Personalized word-guessing game:**
  - The target word is themed around the child's interests (but never the exact word they named).
  - Three clues form a *cueing hierarchy*: broad semantic -> specific feature -> phonemic..
  - A separate LLM call **judges** each guess as `correct`, `incorrect`, or `stop`.
  - After each round the robot **asks the child** whether they want to try another word.
- **Category naming game** (`category_game.py`):
  - The robot names a concrete, familiar category related to the child's likes.
  - An LLM call **judges** each answer as `valid`, `duplicate`, `invalid`, or `stop`.
  - Progressive **hints** are offered after repeated wrong answers.
  - Used categories are remembered in `used_categories.json` so rounds stay varied.
- **Flash card animal game** (`flash_game.py`, `object_detect.py`):
  - The robot asks the child to show an animal; the child holds up a physical card.
  - A **webcam + YOLOv8** model (`object_detect.py`) detects the animal in view.
  - The detected animal is compared to the requested target, with spoken feedback.
  - **spaCy** sentence similarity is used to avoid the robot repeating itself.
- **Animations on every turn:** the model picks a movement keyword (`wave`, `applause`, `think`, `shrug`, `bow`, `celebrate`, `yes`, `no`) which maps to a Blockly behavior or a custom head-motor nod/shake.
- A carefully designed **system prompt** that enforces:
  - Peer-role tone (friend, not teacher),
  - Short, simple sentences,
  - Recasting instead of "no/wrong",
  - Brief 1–2 sentence replies,
  - One question at a time, always ending with a question to keep the conversation going.
- Stand-up at the start of the session and crouch at the end.
- Way to end: The user includes "Bye" in his response.

## Requirements

### Hardware
- An Alpha Mini robot, switched on and connected to the Robots-in-de-Klas platform.
- Pause the default agent on the robot before running the script (open the robot's menu in the portal and click *pause*. The robot will say *"agent is paused"*).
- **A webcam** on the computer running the script. **Required for the flash card game**.
- A small set of printed **animal cards** for the flash card game (or phone photos).

### Software
- **Python 3.10 or higher** (the script uses `list[dict]` type hints).
- An **OpenAI API key**. You can create one at <https://platform.openai.com/api-keys>.
- A **Mistral API key**. *Only if you need to test and edit changing the robot's voice* (experimental, disabled by default). You can create one at <https://admin.mistral.ai/organization/api-keys>.
- The realm string of your specific Alpha Mini (e.g. `rie.69f3564c26d8af1680827d60`). Found on the robot's page in the Robots-in-de-Klas portal.

### Python packages (see `requirements.txt`)
- `autobahn[twisted,serialization]`
- `pyopenssl`
- `alpha_mini_rug`
- `openai`
- `dotenv`
- `mistralai`
- `service_identity`

**Additional packages used by the games (install these too):**
- `spacy` — sentence similarity in the flash card game. Also download the model: `python -m spacy download en_core_web_lg`.
- `ultralytics` — YOLOv8 object detection (`object_detect.py`). The weights file `yolov8n.pt` is downloaded automatically on first run.
- `opencv-python` — webcam capture and the detection preview window (`cv2`).
- `numpy` — image array handling.

## Installation

### 1. Get the project

Clone the repository:

```shell
git clone https://github.com/SarDim99/HRI_coding.git
```

Or place `main.py`, `category_game.py`, `flash_game.py`, `object_detect.py`, and `requirements.txt` together in a folder. Then open a terminal in that folder.

### 2. (Recommended) Create and activate a virtual environment

**Windows (PowerShell):**
```powershell
python -m venv venv
venv\Scripts\Activate.ps1
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install the dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
pip install spacy ultralytics opencv-python numpy
python -m spacy download en_core_web_lg
```

The first run of the flash card game also downloads the `yolov8n.pt` YOLO weights automatically.

## Configuration

### 1. API keys

Create a file called **`.env`** next to `main.py`:

```
OPENAI_API_KEY=sk-your_real_openai_key_here
MISTRAL_TTS_KEY=your_mistral_key_here (optional)
```

`load_dotenv()` at the top of the script picks these up. The `OPENAI_API_KEY` is required. The `MISTRAL_TTS_KEY` is only needed if you enable the experimental Mistral voice (disabled by default).

### 2. Robot realm

In `main.py`, near the bottom, replace the placeholder realm with the realm of your robot:

```python
wamp = Component(
    transports=[{
        "url": "ws://wamp.robotsindeklas.nl",
        "serializers": ["msgpack"],
        "max_retries": 0
    }],
    realm="rie.XXXXXXXXXXXXXXXXXXXXXXXX",  # <-- your robot's realm
)
```

### 3. (Optional) Change the language

The script speaks English by default. To switch to Dutch, change the language code from `"en"` to `"nl"`:

```python
yield session.call("rie.dialogue.config.language", lang="en")  # change "en" to "nl"
```

Note that the prompts are written in English, so for a Dutch session you should also translate `ROLE_PROMPT`, `INTRO_PROMPT`, and the game prompts.

### 4. (Optional) Reset the saved profile

To start fresh with a new child, delete `patient_profile.json` (or empty its contents). The robot will then run the introduction phase from scratch instead of welcoming a returning child. To reset category variety, also delete `used_categories.json`.

## Running the demo

1. Make sure the robot is on, online (green check in the portal), and that its default agent is paused.
2. If you plan to play the flash card game, make sure a webcam is connected and your animal cards are ready.
3. Activate your virtual environment if you created one.
4. Run the script:

```bash
python main.py
```

If everything is configured correctly, the robot will:
1. Stand up and greet the user.
2. Hold an introduction conversation, learning (or recalling) the child's name, age, and interests.
3. Suggest a game. Once the child agrees, the model appends one of `[GUESSING_GAME_START]`, `[CATEGORY_GAME_START]`, or `[FLASH_GAME_START]` and that game begins.
4. After each round the robot asks whether the child wants to play another.
5. Continue until the model includes **"Bye"**, after which the robot says goodbye and crouches.

You can follow the session in the terminal, which logs the recognised speech, the chosen target word/category, and each guess verdict (lines prefixed with `[GAME]`, `[CATEGORY]`, etc.).

### Testing a game on its own

`category_game.py` has a built-in text runner, so you can test the category game logic without the robot:

```bash
python category_game.py
```

## Troubleshooting

- **The robot does not say anything at startup.** Check that it is online, that the default agent is paused, and that the realm string is copied in full.
- **`openai.AuthenticationError` or `Incorrect API key provided`.** Check that you are using the correct API key in `.env`.
- **`ModuleNotFoundError: dotenv`.** Install `python-dotenv`: `pip install python-dotenv`.
- **`ModuleNotFoundError: spacy` / `ultralytics` / `cv2`.** Install the extra game packages: `pip install spacy ultralytics opencv-python numpy`.
- **`OSError: [E050] Can't find model 'en_core_web_lg'`.** Download the spaCy model: `python -m spacy download en_core_web_lg`.
- **The flash card game does not see the card / camera window is black.** Make sure a webcam is connected and not in use by another app; `object_detect.py` opens camera index 0. Good lighting and holding the card steady help YOLO detect it.
- **`msgpack` error.** Run `pip install msgpack`.
- **`TLS failure: certificate verify failed`.** Run `pip install --upgrade pyopenssl`.
- **The game never starts.** The model did not emit a game marker. Make sure the child clearly agrees to play; check the terminal for the `marker detected` line.
- **The same word/category keeps coming up.** Used words are stored in game state for the session; used categories are stored in `used_categories.json`. Delete `patient_profile.json` (and `used_categories.json`) between very short test runs if the variety is too small.
- **The robot greets a stranger by the wrong name.** A previous child's profile is still saved. Delete or clear `patient_profile.json`.
- **Conversation never ends.** The model has not produced a "Bye".
