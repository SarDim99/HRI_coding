# Alpha Mini Chatbot - DLD Get-to-know Conversation Demo

A spoken-dialogue chatbot for the **Alpha Mini** social robot, designed to hold a friendly *getting-to-know-you* conversation with a child who has **Developmental Language Disorder (DLD)**. The robot listens via speech-to-text (STT), sends the recognised text to OpenAI's **GPT-4o-mini**, and speaks the reply back through text-to-speech (TTS).

---

## Features

- Verbal turn-taking dialogue between the robot and a child user.
- Cloud STT through `rie.dialogue.stt.stream` and TTS through `rie.dialogue.say`.
- LLM responses generated by **OpenAI GPT-4o-mini** through the official `openai` Python SDK.
- **Multi-turn memory:** the script maintains a `conversation_history` list so the model remembers earlier turns within the session.
- A carefully designed **system prompt** that enforces:
  - Peer-role tone (friend, not teacher),
  - Short, simple sentences (≤ 12 words),
  - Recasting instead of "no/wrong",
  - Brief 1–2 sentence replies,
  - One question at a time,
  - The goal of learning the child's name, age, and favourite things.
- Stand-up at the start of the session and crouch at the end.
- Two ways to end the conversation:
  - The user says `quit`, `exit`, or `:q`.
  - The model decides to say goodbye and includes the word **"Bye"** in its reply.

## Requirements

### Hardware
- An Alpha Mini robot, switched on and connected to the Robots-in-de-Klas platform.
- Pause the default agent on the robot before running the script (open the robot's menu in the portal and click *pause*. The robot will say *"agent is paused"*).

### Software
- **Python 3.10 or higher** (the script uses `list[dict]` type hints).
- An **OpenAI API key**. You can create one at <https://platform.openai.com/api-keys>.
- The realm string of your specific Alpha Mini (e.g. `rie.69f3564c26d8af1680827d60`). Found on the robot's page in the Robots-in-de-Klas portal.

### Python packages (see `requirements.txt`)
- `autobahn[twisted,serialization]`
- `pyopenssl`
- `alpha_mini_rug`
- `openai`
- `dotenv`

### 1. Get the project

Clone the repository:

```shell
git clone https://github.com/SarDim99/HRI_coding.git
```

Or place `demo.py` and `requirements.txt` together in a folder. Then open a terminal in that folder.

**Windows (PowerShell):**

```bash
python3 -m venv venv
source venv/bin/activate
```

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## Configuration

Two things have to be set up before running the demo: your OpenAI API key and the robot's realm.

### 1. OpenAI API key

Create a file called **`.env`** next to `demo.py` with the following content:

```
OPENAI_API_KEY=sk-your_real_openai_key_here
```

`load_dotenv()` at the top of the script will pick this up.


### 2. Robot realm

In `demo.py`, near the bottom, replace the placeholder realm with the realm of *your* robot:

```python
wamp = Component(
    transports=[{
        "url": "ws://wamp.robotsindeklas.nl",
        "serializers": ["msgpack"],
        "max_retries": 0
    }],
    realm="rie.XXXXXXXXXXXXXXXXXXXXXXXX",  # <-- your robot's realm
)
```

### 3. (Optional) Change the language

The script speaks English by default. To switch to Dutch, change the language code from `"en"` to `"nl"`:

```python
yield session.call("rie.dialogue.config.language", lang="en")  # change "en" to "nl"
```

Note that the system prompt is written in English, so for a Dutch session you should also translate `SYSTEM_PROMPT`.

## Running the demo

1. Make sure the robot is on, online (green check in the portal), and that its default agent is paused.
2. Activate your virtual environment if you created one.
3. Run the script:

```bash
python demo.py
```

If everything is configured correctly, the robot will:
1. Stand up.
2. Greet the user: *"Hello there! It's nice to see you."* and *"You can start a conversation with me whenever you're ready."*
3. Listen, send each utterance (with the running history) to GPT-4o-mini, and speak the reply.
4. Continue until either the user says `quit` / `exit` / `:q`, or the model includes **"Bye"** in its response, after which the robot says goodbye and crouches.

## How it works

| Section | Purpose |
|---|---|
| `load_dotenv()` | Loads the `OPENAI_API_KEY` from a local `.env` file. |
| `SYSTEM_PROMPT` | Long-form instruction that defines the robot's persona, the DLD-friendly speaking rules, and the conversation goals. |
| `chatbot = OpenAI(...)` | Initialises the OpenAI client. |
| `conversation_history` | List of `{role, content}` dicts that grows with every turn, giving the model multi-turn memory. |
| `ask_llm(user_text)` | Appends the user's message, calls `chat.completions.create` with `[system] + history`, stores the assistant reply, and returns it. |
| `asr(frames)` | STT callback. When a `final` frame arrives, the recognised text is stored in `query` and `finish_dialogue` is set to `True`. |
| `main(session, details)` | Sets language, makes the robot stand, greets the user, opens the STT stream, and runs the dialogue loop. |
| Dialogue loop | On each finalised utterance: closes STT, checks for exit keywords, otherwise calls `ask_llm`, speaks the reply, and re-opens STT. Also breaks out when the previous reply contained `"Bye"`. |
| `wamp = Component(...)` + `run([wamp])` | Standard Autobahn/Twisted setup that connects to the Robots-in-de-Klas WAMP server. |

## Troubleshooting

- **The robot does not say anything at startup.** Check that it is online, that the default agent is paused, and that the realm string is copied in full.
- **`openai.AuthenticationError` or `Incorrect API key provided`.** Check if you are using the correct API key.
- **`ModuleNotFoundError: dotenv`.** Install `python-dotenv`: `pip install python-dotenv`.
- **`msgpack` error.** Run `pip install msgpack`.
- **`TLS failure: certificate verify failed`.** Run `pip install --upgrade pyopenssl`.
- **The robot replies to itself / loops.** Echo from the speakers is being picked up by the microphone. Lower the room's reverberation, lower mic sensitivity, or extend the `sleep(1)` after closing the STT stream.
- **Conversation never ends.** The model has not produced a "Bye" yet and the user has not said `quit` / `exit` / `:q`. Either say one of those keywords or steer the conversation toward an ending.
